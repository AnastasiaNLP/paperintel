# REST API Guide

PaperIntel exposes a FastAPI REST API.

Interactive documentation is generated by FastAPI:

- Swagger UI: `http://127.0.0.1:8000/docs`
- OpenAPI schema: `http://127.0.0.1:8000/openapi.json`

This document explains the normal workflow and operational notes. The generated
OpenAPI schema is the source of truth for request and response shapes.

## Start the API

```bash
.venv/bin/python -m dotenv run -- \
  .venv/bin/uvicorn api.rest.main:app --host 127.0.0.1 --port 8000
```

## Workflow

1. Create a session.
2. Analyze a paper URL.
3. Ask questions about analyzed papers.

## Endpoints

| Method | Path | Purpose |
| --- | --- | --- |
| `GET` | `/health` | Check Postgres, Qdrant, and LLM configuration. Returns `503` when degraded. |
| `POST` | `/sessions` | Create a session with persona `engineer`, `researcher`, or `techlead`. |
| `GET` | `/sessions/{session_id}` | Get session state. |
| `GET` | `/sessions/{session_id}/turns` | Get recent conversation turns. |
| `POST` | `/sessions/{session_id}/analyze` | Analyze a paper URL. URL validation happens at the API boundary. |
| `POST` | `/sessions/{session_id}/ask` | Ask a question about active papers in the session. |

## Example

```bash
SESSION_ID=$(
  curl -s -X POST http://127.0.0.1:8000/sessions \
    -H 'content-type: application/json' \
    -d '{"persona":"engineer"}' \
  | python -c "import sys,json; print(json.load(sys.stdin)['id'])"
)

curl -s -X POST "http://127.0.0.1:8000/sessions/$SESSION_ID/analyze" \
  -H 'content-type: application/json' \
  -d '{"paper_url":"https://arxiv.org/abs/1706.03762"}'

curl -s -X POST "http://127.0.0.1:8000/sessions/$SESSION_ID/ask" \
  -H 'content-type: application/json' \
  -d '{"question":"What is the main contribution?"}'
```

## Notes

- `/analyze` is synchronous and can take 50-90 seconds.
- `/ask` works only after a paper has been successfully analyzed and indexed in
  the same session.
- API responses intentionally exclude internal `AgentRun` payloads and raw
  structured errors. Those are stored for observability, not returned as public
  transport data.
- Invalid paper URLs return `422`.
- Missing sessions return `404`.
- Unexpected service failures return a safe `500` response.
